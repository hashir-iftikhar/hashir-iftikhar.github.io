A fully automated AI-powered analysis system built around Formula 1 race data. The pipeline pulls live data from the Jolpica-F1 API, uses Claude (Sonnet for analysis, Haiku for content generation) to generate insights, stores results in Supabase, and automatically publishes to Facebook, Discord, Gmail, and Telegram — all orchestrated through n8n with zero manual intervention.

The system runs on two branches: a race-day flow that triggers full race analysis and multi-platform publishing, and a non-race-day flow that shares curated F1 facts instead — keeping engagement consistent even between race weekends.

Tech:
  • n8n
  • Claude API
  • Supabase
  • Webhook Automation
  • Multi-Platform Publishing

A structured data warehouse built in SQL Server using the Medallion Architecture pattern — organizing raw sales data through Bronze (raw), Silver (cleaned/transformed), and Gold (business-ready) layers. The project focused on building reliable ETL pipelines and resolving real data quality issues, including a product key normalization mismatch discovered between Silver-layer tables.

The final Gold-layer views are designed to directly support business reporting and analytics use cases, with the full technical process documented in an IEEE-format two-column report.
